Overview

Policy ID

All assets on the Cardano blockchain are created in the context of a policy and its associated Policy ID.
Assets (e.g. a learner achievement) can have the same name.
It is the combination of the Policy ID and the Asset Name that make an asset unique.
selfdriven Policy IDs.

Assets

The tokens and associated metadata stored on the Cardano blockchain.
e.g. an achievement (as an asset) is stored on-chain as a SDA token with metadata describing the achievement.
selfdriven Tokens.
Example SDA Metadata On-Chain.

Getting Assets On-Chain

The selfdriven cloud service periodically analyses the new data entered via the App or API (e.g. a validated achievement with skills) and when applicable coverts it into a Cardano asset - it then submits the asset as a transaction to the Cardano blockchain.

Accessing On-Chain Assets

Assets (tokens/metadata) can be accessed by the App or by directly querying the Cardano blockchain using a service like blockfrost and the selfdriven SDK.
Non identity based (fungible) tokens like the selfdriven Community (SDC) token can also be accessed directly using a Cardano wallet.
